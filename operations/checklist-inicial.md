# Checklist Inicial

## Inicio de ciclo

- revisar o objetivo atual do projeto
- atualizar o contexto em `docs/persona-e-contexto.md` quando necessario
- registrar decisoes importantes no repositorio

## Durante o trabalho

- manter arquivos e nomes consistentes
- evitar subir dados sensiveis
- separar documentacao de material temporario

## Fechamento

- revisar o que mudou
- commitar com uma mensagem clara
- sincronizar com o GitHub
- validar que o repositorio local e o remoto seguem alinhados
